## 3. 协调Claude Code会话团队

**原文标题**: Orchestrate teams of Claude Code sessions

**原文链接**: [https://code.claude.com/docs/en/agent-teams](https://code.claude.com/docs/en/agent-teams)

**Claude Code 中的智能体团队功能概述**

智能体团队功能支持多个 Claude Code 实例之间的协同工作，其中一个会话作为主导者协调整体工作。与仅向主智能体汇报的子智能体不同，团队成员在各自独立的上下文窗口中独立运行，并且可以直接相互通信。

**核心特性：**
*   **最佳适用场景：** 研究、调试不同理论假设或跨层开发（前端/后端/测试）等并行任务。
*   **设置方式：** 在设置中启用 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`。
*   **控制方式：** 使用自然语言指示主导者来管理团队、分配任务和综合结果。
*   **显示模式：** 可选择“进程内”（所有内容在一个终端中）或“分屏模式”（需要 tmux/iTerm2）。
*   **直接交互：** 您可以直接向任何团队成员发送消息以指导其工作。
*   **任务管理：** 共享任务列表协调工作，任务之间可设置依赖关系。
*   **计划审批：** 对于复杂工作，可要求团队成员在实施前提交计划供主导者审批。
*   **委派模式：** 限制主导者仅执行协调任务，防止其自行实施具体工作。

**重要说明：**
*   团队模式消耗的令牌数显著高于单个会话。
*   务必使用主导者来清理团队；团队成员不应运行清理操作。
*   最佳实践包括：在生成团队成员时提供清晰的上下文提示，并合理规划任务规模以平衡生产力和协调开销。

## 5. 别租云，要拥有云。

**原文标题**: Don't rent the cloud, own instead

**原文链接**: [https://blog.comma.ai/datacenter/](https://blog.comma.ai/datacenter/)

在这篇博客文章中，comma.ai的首席技术官Harald Schäfer主张企业应自建数据中心，而非依赖云服务提供商。他认为云服务会导致供应商锁定、高昂成本，并助长低效的工程实践。相比之下，自主运营计算资源能培养自主能力、提升实用工程技能，并显著节约成本——comma估算其自建方案仅花费500万美元，而若采用云服务预计将超过2500万美元。

文章详细介绍了comma的内部数据中心：该中心功耗约450千瓦，采用室外空气冷却；核心计算设备由75台定制TinyBox Pro机构成，配备600块GPU；存储采用戴尔服务器，通过SSD承载约4PB数据；网络基于简化的100Gbps以太网和Infiniband交换机搭建。

关键软件基础设施包括：
*   **Minikeyvalue (mkv)**：用于高速数据访问的分布式存储系统
*   **Slurm**：工作负载与计算节点管理工具
*   **PyTorch**：结合FSDP用于分布式模型训练
*   **Miniray**：轻量级开源并行计算任务调度器
*   **基于NFS的单一代码库**：确保所有分布式工作的代码一致性

文章最后展示了这些组件如何通过单条命令集成运行复杂的策略驾驶模型训练，印证了这种自建方案带来的高效性与可控性。

## 25. 编程模式：提花织机的故事

## 26. Triton定制布局

**原文标题**: Triton Bespoke Layouts

**原文链接**: [https://www.lei.chat/posts/triton-bespoke-layouts/](https://www.lei.chat/posts/triton-bespoke-layouts/)

本文解释了Triton的“定制布局”（亦称传统布局），这是专为GPU编程中特定硬件模式设计的张量布局。文章将其与更新、更通用的“线性布局”进行对比，指出定制布局对开发者更直观，但对编译器优化的灵活性较低。

文章详述了定制布局的两大类别：

1.  **分布式布局：** 这类布局模拟张量元素如何被不同硬件单元（线程、线程束、CTA）持有。
    *   **分块布局：** 用于SIMT模型中实现高效的全局内存访问。它规定每个线程持有的元素数量以及线程/线程束的排列方式，通过`order`属性定义变化最快的维度。
    *   **MMA布局：** 供应商特定的布局（如AMD MFMA），描述为张量/矩阵计算单元（例如矩阵乘法中的C矩阵）排列数据的方式。
    *   **点积操作数布局：** 描述输入张量核心操作的A、B操作数的布局，由父MMA布局派生而来。

2.  **共享布局：** 这类布局模拟数据在共享内存（CTA内所有线程可见）中的排列方式，主要通过**交错重排**（对索引进行按位异或操作）或填充来避免存储体冲突。

文章说明这些布局作为编码属性附加在MLIR张量类型上。关键编译器通道（如用于内存访问的`Coalesce`和用于计算的`AccelerateMatmul`）决定这些布局的选择，而`RemoveLayoutConversions`通道则优化布局间的转换。虽然线性布局为编译器提供了更统一的基础，但定制布局对于开发者直接控制及表示硬件特定模式仍然至关重要。

《简明Scheme：计算机科学导论》（1999）是由布莱恩·哈维和马修·赖特合著的教科书，旨在通过Scheme编程语言教授计算机科学基础概念。全书分为七个部分，从核心编程思想逐步过渡到更高级的主题。
