## 10. 将你的SSH密钥存入TPM芯片中。

**原文标题**: Put your SSH keys in your TPM chip

**原文链接**: [https://raymii.org/s/tutorials/Put_your_SSH_keys_in_your_TPM_chip.html](https://raymii.org/s/tutorials/Put_your_SSH_keys_in_your_TPM_chip.html)

本文阐述了如何将SSH私钥存储在计算机内置的TPM（可信平台模块）芯片中以增强安全性，作为替代便携式硬件安全模块（如YubiKey）或将密钥保存在文件系统中的方案。

作者指出，虽然TPM的安全性低于便携式硬件安全模块（因其始终存在于设备中且无需物理接触），但相比将私钥存储在`~/.ssh/`目录中，其安全性显著更高——密钥永远不会离开硬件模块，从而免受恶意软件和文件系统暴露的威胁。

本指南为Linux用户提供了逐步操作说明：
1. 安装必要的TPM软件工具。
2. 在TPM内创建持久的PKCS#11令牌。
3. **导入现有SSH密钥**（为防范BIOS更新等可能清除TPM数据的风险，建议离线生成备份密钥）至令牌，而非由TPM直接生成密钥。
4. 通过PKCS#11接口配置SSH使用TPM存储的密钥，访问时需要用户PIN码。

此流程最终实现SSH认证时私钥运算在TPM内部完成，同时密钥也可添加至`ssh-agent`以实现基于会话的PIN码输入。
