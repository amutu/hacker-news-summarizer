## 2. 为什么TUI界面正在回归

**原文标题**: Why TUIs Are Back

**原文链接**: [https://wiki.alcidesfonseca.com/blog/why-tuis-are-back/](https://wiki.alcidesfonseca.com/blog/why-tuis-are-back/)

本文认为，终端用户界面（TUI）正因主流操作系统原生桌面UI框架的失效而复兴。在Windows上，微软反复更迭众多框架（MFC、WinForms、WPF、Silverlight、UWP、MAUI），却始终未能实现一致或连贯的应用设计，导致许多人转而依赖Electron应用。在Linux上，GTK与Qt工具包虽在视觉上更具兼容性，但其碎片化问题阻碍了企业开发原生应用。在macOS上，苹果已摒弃曾备受推崇的人机界面指南，破坏了一致性与可用性。与此同时，Electron应用虽受欢迎，却存在内存管理糟糕、缺乏键盘驱动工作流以及视觉不一致等问题。

TUI提供了一种解决方案：它们运行快速、易于自动化、跨平台兼容，并且可远程操作。由于原生工具包未能提供统一的体验，开发者正重返命令行，寻求可靠且无干扰的交互方式。

文章总结道，行业必须回归基本UI原则（引用尼尔森、诺曼和约翰逊的观点），将用户设计视为核心工程技能，并优先构建直观、同质的界面。编程或可自动化，但理解用户需求始终不可或缺。

## 5. OpenAI的o1模型对急诊患者的正确诊断率为67%，而分诊医生的正确诊断率为50-55%。

**原文标题**: OpenAI's o1 correctly diagnosed 67% of ER patients vs. 50-55% by triage doctors

**原文链接**: [https://www.theguardian.com/technology/2026/apr/30/ai-outperforms-doctors-in-harvard-trial-of-emergency-triage-diagnoses](https://www.theguardian.com/technology/2026/apr/30/ai-outperforms-doctors-in-harvard-trial-of-emergency-triage-diagnoses)

《科学》杂志发表的一项哈佛研究发现，OpenAI的o1推理模型在急诊分诊诊断中的表现优于人类医生。当使用标准电子健康记录时，该AI正确诊断了67%的急诊患者，而人类医生的准确率为50%至55%。其优势在信息量极少的高节奏分诊场景中最为显著。随着临床细节增加，AI准确率升至82%，专家准确率为70%至79%——尽管这一差距在统计学上并不显著。在长期治疗方案上，AI得分更高（89%对人类的34%）。

研究人员强调，AI并非要取代医生，而是将重塑医学，形成涉及医生、患者和AI的"三方诊疗模式"。研究指出，AI能够识别人类遗漏的细微线索，例如狼疮与肺部炎症的关联。但该测试仅涉及文本数据；AI无法评估患者痛苦等视觉线索。关于问责机制、过度依赖AI以及可能影响特定患者群体的偏见等问题仍令人担忧。尽管近五分之一的美国医生已在使用AI进行诊断，但专家警告称，该技术在常规临床使用中的安全性尚未得到验证。

## 6. 为一人打造的桌面

**原文标题**: A desktop made for one

**原文链接**: [https://isene.org/2026/05/Audience-of-One.html](https://isene.org/2026/05/Audience-of-One.html)

作者描述了自己历经数周搭建完整定制桌面计算环境的过程，用个人设计的软件取代了沿用二十五年的现成工具。该系统包含两层架构：CHasm（纯x86_64汇编，无libc）处理像素渲染、按键输入等底层功能，而Fe₂O₃（Rust）则基于名为crust的共享TUI库提供应用级工具。

关键替代方案包括：tile（窗口管理器）、glass（终端模拟器）、scribe（文本编辑器）、pointer（文件管理器）、kastrup（邮件/RSS/聊天）、tock（日历）和astro（天文面板）。仅保留Firefox和WeeChat作为第三方工具。

作者特别提到从自2001年开始使用的Vim切换至自研编辑器scribe时那段情感历程。尽管肌肉记忆根深蒂固，切换过程仅用三天。scribe保留了Vim的模态编辑功能，但删除了90%的闲置功能，并新增了个人所需特性：软换行、阅读模式、AI集成及持久化共享寄存器。

本文强调，构建定制工具已不再是需要耗时数年的英雄壮举。借助Rust、AI辅助（Claude Code）及文档完善的TUI编程技术，从产生功能需求到实现它，如今只需投入数个夜晚专注工作。作者明确表示这些软件不面向公众——它们专为一人设计，免除了配置负担、文档编写和功能争执。核心信息是：自建工具正变得日益可行且极具满足感。

**摘要：**

梅赛德斯-奔驰宣布将在未来车型中恢复关键功能的物理按键，以回应客户反馈——触控式操作和深埋于菜单中的功能不够实用。销售总监马蒂亚斯·盖森坦言，两年前就有买家表示“想法很好，但对我们而言不实用”，这促使品牌转向更“模拟化”的设计方案。

## 10. 不良连接：隐蔽监控行为体对全球电信的利用

**原文标题**: Bad Connection: Global telecom exploitation by covert surveillance actors

**原文链接**: [https://citizenlab.ca/research/uncovering-global-telecom-exploitation-by-covert-surveillance-actors/](https://citizenlab.ca/research/uncovering-global-telecom-exploitation-by-covert-surveillance-actors/)

**摘要：**

本报告由Citizen Lab联合电信安全公司发布，详细披露了两起利用全球移动网络漏洞的隐蔽监控行动。这两起行动均归因于支持国家情报活动的商业监控供应商（CSV），通过电信信令协议（3G网络的SS7协议及4G/5G网络的Diameter协议）针对高价值目标实施监控。

**主要发现：**

1. **多载体监控：** 攻击者将3G/4G信令网络攻击与短信设备直接利用相结合——通过包含隐藏SIM卡指令的恶意短信提取位置数据，将目标设备转化为隐蔽追踪信标。

2. **全球基础设施滥用：** 攻击利用了来自17个国家/地区（包括以色列、英国、中国、泰国、瑞典等）的运营商标识，展现了广泛的国际渗透范围。

3. **系统性漏洞：** 攻击利用了电信协议中重效率轻安全的设计缺陷。运营商普遍未实施Diameter协议的安全功能，使其延续了与传统SS7协议相同的信任型漏洞。

4. **持续性活动：** 行动方多年重复使用相同运营商标识，表明长期监控行动持续运行。对互联流量的审查薄弱使攻击者得以通过可信运营商路径实施攻击。

5. **可及性：** 第三方服务商（漫游枢纽、消息服务）为攻击者提供了间接接入信令生态系统的渠道，将潜在攻击者范围扩大到移动运营商之外。

报告指出，尽管已有多次公开披露，此类监控行动仍未收敛，引发对电信行业问责机制、监管体系及全球安全格局的深切担忧。
