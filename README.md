## 6. 第五范式与数据库设计

**原文标题**: 5NF and Database Design

**原文链接**: [https://kb.databasedesignbook.com/posts/5nf/](https://kb.databasedesignbook.com/posts/5nf/)

本文批判了数据库设计中关于第五范式（5NF）的传统解释，认为它们不必要地令人困惑。文章提出了一种更清晰、更具逻辑性的设计流程：从业务需求出发，创建实体（锚点）和关系（链接）的逻辑模型，然后应用标准的表设计策略来实现规范化的物理模式。

作者通过分析两个常见的5NF教学示例来阐明这一方法：
1.  **“冰淇淋”示例（AB-BC-AC三角关系）：** 涉及三个实体（品牌、口味、朋友），每对实体之间都存在多对多关系。逻辑模型自然地导出三个独立的联结表，避免了单一三列表可能产生的冗余。
2.  **“音乐家”示例（ABC+D星型模式）：** 涉及记录哪位音乐家在哪个音乐会上演奏了哪种乐器。分析揭示了一个潜在的第四实体——“表演”，它与其他三个实体相关联。这产生了一个包含外键的单一表，其中是选择合成主键还是复合主键取决于业务唯一性规则。

核心结论是：你无需死记硬背5NF规则。通过严格遵循逻辑建模流程——识别所有实体并精确定义它们之间的关系——你将自然而然地得到一个规范化良好、无异常的数据库设计。

**原文链接**: [https://www.theregister.com/2026/04/14/eff_california_3dprinted_firearms/](https://www.theregister.com/2026/04/14/eff_california_3dprinted_firearms/)

一项拟议的加州法案（AB 2047）将要求3D打印机制造商安装经州认证的软件，该软件可扫描枪支部件的设计文件并阻止其打印。尽管该法案旨在遏制无法追踪的“幽灵枪”，但数字权利组织电子前沿基金会（EFF）认为该法律存在严重缺陷。

EFF活动家克里夫·布劳恩和罗里·米尔指出，这项规定在技术上不可行，因为用户可以轻易修改设计以规避检测。他们警告，这实际上将禁止开源3D打印软件，迫使用户依赖专有系统，并导致打印活动受到广泛监控。他们担心，这种限制可能从枪支扩展到封锁受版权保护的玩具或维修零件等物品。

该法案还遭到加州枪支持有者协会的反对，他们认为这给合法用户带来负担，而非针对犯罪分子。Prusa Research等3D打印公司则担忧，该法律违背开源原则和用户自主权。

总之，该立法试图解决幽灵枪问题，但因其可能抑制创新、侵犯隐私、技术效果有限并惩罚普通消费者而受到批评。
