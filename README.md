## 3. 诱饵字体

**原文标题**: Decoy Font

**原文链接**: [https://www.mixfont.com/experiments/decoy-font](https://www.mixfont.com/experiments/decoy-font)

**Decoy 字体简介**

Decoy 是一款免费可下载的 TTF 字体，采用混合图像技术，可向 AI 隐藏信息，同时保持人类可读。每个字母编码两个不同字符：一个细线高频率的前景轮廓（诱饵）与一个模糊的低频率背景形状（隐藏信息）。当近距离查看或由分析像素的 AI 系统（如 ChatGPT、GPT Sol 或 Gemini）读取时，模型将读取前景诱饵文本；然而当人类退后或眯眼时，模糊的背景信息便会清晰显现。

该字体运用了著名视错觉（例如爱因斯坦与玛丽莲·梦露合二为一的图像）中相同的空间频率原理。它被设计为一种"反 AI"工具，用于保护文本免遭抓取或 OCR 识别。文章指出，虽然 Decoy 字体能够迷惑 AI，但并非万无一失——具备编程或提示能力的高级智能体仍可能解码隐藏信息。

Decoy 是 Mixfont 实验的一部分（与利用动态效果的 Ghost 字体并列），以标准 TTF 文件形式提供，可直接用于项目。潜在应用包括验证码、私密通信及 AI 文本识别基准测试。创作者欢迎反馈，并建议扩展该字体以支持中文等表意文字。

**原文标题**: Detecting LLM-Generated Texts with “Classical” Machine Learning

**原文链接**: [https://blog.lyc8503.net/en/post/llm-classifier/](https://blog.lyc8503.net/en/post/llm-classifier/)

**摘要：**  
作者使用经典机器学习（scikit-learn）构建了一个AI生成文本检测器，并推测其原理与商业"AI查重工具"类似。  

**关键步骤：**  
- **数据：** 2022年前网文及同人作品中的人类文本（约1万份样本）；通过低成本API技巧调用七种不同大语言模型（Gemini、Qwen、GLM、DeepSeek等）生成的AI文本。  
- **方法：** 句子级TF-IDF → LinearSVC + 七个二元分类器的多数投票机制。  
- **性能：** 单句准确率约85%；在60%阈值下，整篇文章的误报率低于0.04%。  

**演示：** 网页版完全在JavaScript中运行TF-IDF + SVM（模型大小107MB，处理100万字符约需10秒）。对已知及未知大模型（如Claude、GPT-5）的检测准确率均超70%。  

**规避尝试：** 翻译和改写提示仅能轻微降低检测率（79–86%），证明了该方法的鲁棒性。  

**结论：** 大语言模型输出中的统计模式足够显著，即使简单的TF-IDF + SVM也能可靠区分AI与人类写作，且对人类文本的误报率近乎为零。

## 9. 我们的Rust到Zig重写进展如何

**原文标题**: How Our Rust-to-Zig Rewrite Is Going

**原文链接**: [https://rtfeldman.com/rust-to-zig](https://rtfeldman.com/rust-to-zig)

文章详细介绍了Roc编译器团队历经487天将30万行Rust代码重写为Zig，近期实现了功能对等。核心动因包括：Rust编译速度慢、需要细粒度内存分配器（Zig的优势），以及对内存不安全代码的更好支持（编译器需要大量不安全操作）。重写实现了热代码加载（零分配HTTP路由）、交叉编译为静态二进制文件，以及带有字符串插值的高级模式匹配等新功能。

作者认为，虽然Rust的借用检查器提供了编译时安全性，但Zig的ReleaseSafe运行时检查能在开发阶段捕获释放后使用错误，而原Rust代码中有1200处`unsafe`使用——这使得Zig的方案对编译器开发更实用。该项目还修复了架构问题，尤其是在lambda集合特化（去函数化）方面，这本身就是重写的原始原因。

总体而言，重写提升了性能（WASM二进制文件从旧编译器的69KB+缩减至31KB），并在更快的构建速度、更好的内存控制以及与编译器特定需求相关的生态适应性方面达到了预期。作者指出，相较于Bun的11天从Zig到Rust的重写，本项目耗时较长是由于范围不同（前者是直接移植，而后者是完整的架构重新设计）。
