主要功能包括多难度级别（从单音到复杂和弦逐步进阶）、简洁直观的界面设计以及即时听觉反馈。该项目以“Show HN”帖文形式发布，邀请社区体验游戏，并就其作为学习工具的理念与实现提供反馈。

**《科学家在六万年前箭头上发现最古老毒药》摘要**

南非一项突破性考古发现揭示了人类在狩猎工具上使用毒药的最早已知证据。研究人员分析了在该国南部海岸附近洞穴中发现的一批六万年前的骨制箭头。

关键发现在于检测到这些古老器物上残留的有毒化学物质。科学分析识别出来自蓖麻植物的有毒化合物痕迹，这表明该地区的旧石器时代晚期居民不仅制作了精密的骨尖投掷武器，还使用了植物性毒素以增强其杀伤力。

研究结果表明，这些早期人类极具智慧，他们运用多部件技术（骨制箭尖、木制箭杆、粘合剂和毒药）来提升狩猎优势，可能因此能更高效地捕杀大型猎物。这一发现强调了非洲人类社会在农业出现之前很久就已具备的技术复杂性。

## 7. 如何用26字节存储一个棋局位置（2022）

**原文标题**: How to store a chess position in 26 bytes (2022)

**原文链接**: [https://ezzeriesa.notion.site/How-to-store-a-chess-position-in-26-bytes-using-bit-level-magic-df1fdb5364eb42fdac11eb23b25e9605](https://ezzeriesa.notion.site/How-to-store-a-chess-position-in-26-bytes-using-bit-level-magic-df1fdb5364eb42fdac11eb23b25e9605)

本文介绍了一种仅用26字节存储国际象棋棋局的方法，比标准的70字节FEN表示法更高效。

其核心在于仅存储必要信息：棋子位置、行棋方、王车易位权及吃过路兵目标格。该方法省略了回合计数，因为这些信息并非定义棋盘状态所必需。

该技术采用**位棋盘**，即通过多个64位整数中的单个比特来表示64个棋盘格。一组位棋盘标识棋子类型（兵、马等），另一组标识棋子颜色。作者通过巧妙的比特打包与压缩技术（如对棋子类型采用霍夫曼编码），将总数据量压缩至26字节。

存储结构概要如下：
*   13字节用于棋子位置位棋盘
*   1字节用于行棋方与王车易位权
*   1字节用于吃过路兵目标格（若存在）
*   11字节用于霍夫曼编码树结构开销

这种紧凑表示法适用于游戏数据库、AI训练或网络传输等对存储空间与带宽要求严苛的应用场景。

## 10. 展示 HN：多种形状正则化算法

**原文标题**: Show HN: Various shape regularization algorithms

**原文链接**: [https://github.com/nickponline/shreg](https://github.com/nickponline/shreg)

**shreg** 是一个用于形状正则化的 Python 库，该技术旨在清理有噪声或不精确的几何数据。它提供多种算法来对齐线段并简化轮廓，从而生成更干净、更规则的形状，适用于 CAD、网格生成和计算几何。

主要功能包括：
*   **线段正则化：** 使用二次规划对齐线段的角度和位置。
*   **端点吸附正则化：** 连接邻近的端点以形成封闭多边形。
*   **度量正则化：** 强制执行等长、长度量化和等间距等约束。
*   **轮廓正则化：** 通过将边对齐到主方向来简化闭合多边形。
*   **T 型连接点检测：** 将端点吸附到其他线段的内部。

该库的核心功能无外部依赖，并包含可视化工具。它将正则化问题表述为通过二次规划求解的能量最小化问题，在保持对原始数据的保真度与几何规则性之间取得平衡。应用场景包括建筑图纸清理、从草图生成干净网格以及矢量图形简化。
