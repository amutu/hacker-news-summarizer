**原文标题**: California's New Bill Requires DOJ-Approved 3D Printers That Report Themselves

**原文链接**: [https://blog.adafruit.com/2026/02/19/californias-new-bill-requires-doj-approved-3d-printers-that-report-on-themselves/](https://blog.adafruit.com/2026/02/19/californias-new-bill-requires-doj-approved-3d-printers-that-report-on-themselves/)

**加州新法案要求配备可自身上报的司法部认证3D打印机摘要**

加州新提案SB 1034法案拟对能够制造枪支或枪支零部件的3D打印机实施严格监管，核心要求包括：

1.  **司法部认证**：所有在州内销售的3D打印机需获得加州司法部批准。
2.  **序列化与上报机制**：每台打印机须配备唯一序列号及“自身上报”功能，需记录所有打印物品并将数据连同打印机序列号、位置信息传输至司法部。
3.  **制造商注册**：相关打印机生产企业须向司法部注册并缴纳年费。
4.  **违禁品限制**：明确禁止使用此类打印机生产无序列号的枪支、枪支零部件或其他武器，违者可能构成轻罪。

该法案旨在遏制无法追踪的“幽灵枪”扩散，但已引发制造者与3D打印社区的强烈批评。反对者认为法案范围过宽，实质上将所有3D打印机视为潜在制枪工具，将带来沉重的技术负担、高昂成本及隐私隐患。批评者还指出，此举可能抑制创新、损害小型企业，并妨碍该技术在原型设计、艺术创作和工具制造等合法领域的应用。

文章指出，这项充满争议的法案虽试图解决幽灵枪问题，但可能对加州更广泛的3D打印产业及爱好者造成深远的负面影响。

## 10. 告别Rust

**原文标题**: Farewell Rust

**原文链接**: [https://yieldcode.blog/post/farewell-rust/](https://yieldcode.blog/post/farewell-rust/)

本文详述了作者从在学校学习Pascal和C语言，到最终以独立创始人身份使用Rust构建Web应用的历程。作者最初虽被Rust的低层控制能力和现代工具链所吸引，但最终决定将项目迁移至Node.js。

这一转变的核心原因源于Web开发的特定需求。作者发现Rust生态在关键领域存在不足，例如缺乏兼具强类型安全的成熟模板引擎、完整的国际化（i18n）支持，以及易于使用的第三方API库。编译时间在本地尚可管理，但在CI/CD流水线中却成为显著瓶颈，拖慢了修复生产环境问题的关键反馈循环。

此外，Web开发动态特性（涉及频繁的数据序列化和动态SQL查询）导致Rust中需要编写大量样板化的错误处理代码。相比之下，Node.js尽管存在JavaScript的固有特性，却提供了更成熟的生态工具链（如TypeScript、Zod、Kysely），既能保障足够的类型安全，又实现了大幅加快的部署周期。

作者总结道：虽然Rust在性能关键的非可视化领域表现出色，但对于独立Web开发而言——迭代速度和生态成熟度至关重要——Node.js成为了更高效且“足够好”的选择，即便这意味着需要牺牲部分语言层面的优雅性。
