**原文链接**: [https://health.ucdavis.edu/news/headlines/first-ever-in-utero-stem-cell-therapy-for-fetal-spina-bifida-repair-is-safe-study-finds/2026/02](https://health.ucdavis.edu/news/headlines/first-ever-in-utero-stem-cell-therapy-for-fetal-spina-bifida-repair-is-safe-study-finds/2026/02)

加州大学戴维斯分校健康中心的研究人员安全完成了一项开创性临床试验的第一阶段，成功实施了全球首例子宫内脊柱裂干细胞疗法。研究结果发表在《柳叶刀》杂志上，显示在标准胎儿手术中植入人胎盘来源的干细胞补片未引发任何安全问题。

脊柱裂是一种脊髓未能闭合的先天缺陷，可能导致终身残疾。CuRe试验旨在探究干细胞能否安全增强胎儿手术的再生效果。手术过程中，医生在闭合脊柱缺陷前，将干细胞补片直接覆盖在暴露的脊髓上。

鉴于良好的安全性表现，美国食品药品监督管理局已批准该研究进入下一阶段，将招募最多35名患者。研究人员将对患儿进行为期六年的追踪随访，以评估长期安全性及运动能力、膀胱和肠道功能的潜在改善。

这项由加州再生医学研究所和Shriners儿童医院资助的试验，标志着再生性胎儿疗法的重要转变——其目标不仅是修复，更是促进发育中组织的愈合。

## 8. 使用tmux和Markdown规范进行并行编码代理

**原文标题**: Parallel coding agents with tmux and Markdown specs

**原文链接**: [https://schipper.ai/posts/parallel-coding-agents/](https://schipper.ai/posts/parallel-coding-agents/)

本文介绍了一种轻量级系统，用于通过tmux和基于Markdown的“功能设计”（FD）规范来管理并行编码代理。核心工作流程涉及三个代理角色：规划者（设计规范）、工作者（实现代码）和项目经理（管理待办事项）。

该系统以编号的FD文件为核心，这些文件记录了问题、考虑的解决方案、实施计划和验证步骤。这些文件经历八个阶段（从“已规划”到“已关闭”），并在中央索引中跟踪。六个自定义斜杠命令（例如`/fd-new`、`/fd-status`、`/fd-verify`）自动化了FD的生命周期。

作者在多个终端中使用tmux会话来同时运行多个代理，实际限制为8个代理以保持决策质量。关键支持元素包括项目特定的开发指南（用于编码规范）、用于快速导航的bash别名，以及tmux视觉提示（用于指示代理何时空闲并等待输入）。

该系统通过归档的FD文件创建了有价值的决策轨迹，帮助代理了解过往工作，并结构化验证过程。主要挑战在于高并行度下的认知负荷，以及复杂规划会话中的上下文窗口限制。

## 9. 将Gleam应用打包成单一可执行文件

**原文标题**: Packaging a Gleam app into a single executable

**原文链接**: [https://www.dhzdhd.dev/blog/gleam-executable](https://www.dhzdhd.dev/blog/gleam-executable)

本文介绍了如何将Gleam应用程序打包成单一可执行文件，涵盖其Erlang和JavaScript编译目标的实现方法。

针对**Erlang目标**，主要工具是**Gleescript**，它可以创建escript可执行文件，但要求目标系统已安装Erlang虚拟机。文章还提及**Burrito**作为创建不依赖Erlang的独立可执行文件的潜在方案，但注明该方法尚未在Gleam中经过测试。

针对**JavaScript目标**，评估了多种打包编译工具：
*   **Deno compile**：需先将Gleam的JavaScript输出打包为单个文件（例如使用esbuild），再调用Deno内置编译器。该方法操作直接但生成的可执行文件体积较大。
*   **Node SEA（单文件可执行应用）**：一项实验性Node.js功能。流程复杂，涉及代码打包、创建配置数据块并注入Node二进制文件。作者测试时遇到了段错误问题。
*   **Bun build --compile**：该方法被强调为最便捷快速的方案，通过一条简单命令即可同时完成打包和编译，但生成的可执行文件同样体积庞大。
*   **Nexe**：作为另一款Node.js打包工具被提及，但未展开详述。

作者总结认为，尽管运行时打包的JavaScript可执行文件普遍存在体积较大的问题，但**Bun**在速度与易用性方面提供了最佳组合方案。
