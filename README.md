**原文标题**: FreeInk: Open ecosystem for e-readers

**原文链接**: [https://freeink.org/](https://freeink.org/)

**《“FreeInk：为电子阅读器打造开放生态系统”摘要》**

本文介绍了**FreeInk**，一个旨在对抗亚马逊Kindle和Kobo等专有电子阅读器平台局限性的拟议开放生态系统。其核心理念是赋予用户对其设备和内容的完全控制权。

关键要点包括：

- **开放硬件与软件：** FreeInk倡导基于开源固件（类似于KOReader等基于Linux的项目）和允许第三方修改、应用安装及自定义界面的硬件规格来构建电子阅读器。
- **图书馆自由：** 用户可阅读来自任何来源（本地文件、公共图书馆、独立书店）的无DRM图书，消除供应商锁定，同时通过标准化插件支持可选的DRM。
- **社区驱动开发：** 该生态系统将依赖社区贡献进行更新、功能开发和优化——从而避免封闭系统中常见的制造商弃用问题。
- **隐私与伦理：** FreeInk强调无追踪、无强制更新和透明的数据政策，与基于广告或数据收集的模式形成鲜明对比。

该倡议呼吁制造商、开发者和读者合作，共同创建一个模块化、可维修且可持续的电子阅读器标准。它将自己定位为一种针对阅读体验的数字权利运动，旨在复制其他硬件类别中开源成功的案例。

**原文标题**: Long presumed dead, a thriving coral reef is discovered in West Africa

**原文链接**: [https://e360.yale.edu/digest/benin-coral-reef](https://e360.yale.edu/digest/benin-coral-reef)

**概要：**

一份20世纪60年代的渔业调查报告首次提及贝宁海域存在珊瑚礁，随后便被遗忘。六十年后，由热拉尔·津津多胡率领的贝宁科学家团队重新发现了这片生机勃勃的珊瑚礁。该珊瑚礁位于距海岸14英里、水深超过175英尺处，是一个中层珊瑚生态系统，包含至少8种珊瑚和8种鱼类，包括金色非洲笛鲷和几内亚神仙鱼。

杰克·多西发布Buzz：Block旗下开源协作平台，集合团队聊天、AI代理与Git托管于一体。该工具基于可自托管的Nostrelay构建，通过加密签名事件为人类员工与AI代理提供统一身份架构——包含密钥对、频道成员资格和审计追踪。其设计旨在通过将讨论、代码仓库、工作流自动化及代理活动整合至单一身份系统，减少Block对Slack和GitHub的依赖。

## 6. 苹果因未扫描iCloud中的CSAM内容而免于承担责任

**原文标题**: Apple defeats liability for not scanning iCloud for CSAM

**原文链接**: [https://blog.ericgoldman.org/archives/2026/07/apple-defeats-liability-for-not-scanning-icloud-for-csam-but-the-judge-was-not-pleased-amy-v-apple.htm](https://blog.ericgoldman.org/archives/2026/07/apple-defeats-liability-for-not-scanning-icloud-for-csam-but-the-judge-was-not-pleased-amy-v-apple.htm)

**概要：** 联邦法院驳回针对苹果的诉讼，裁定《通信规范法》第230条保护苹果无需为未扫描iCloud中的儿童性虐待材料（CSAM）承担责任。原告CSAM受害者主张，苹果的设计缺陷——未采用PhotoDNA或自有NeuralHash等工具——导致其受虐图像传播。法官认为，任何扫描义务都将要求苹果充当第三方内容的“发布者”，从而触发第230条豁免权。

法院驳回了基于举报系统或内容修改的例外情形，指出苹果并未创建或修改CSAM。同时裁定，若苹果从未识别相关材料，其亦无义务向国家失踪与受虐儿童中心（NCMEC）报告CSAM。

法官表达强烈不安，称现行法律优先保护隐私而非CSAM受害者，并呼吁立法者强制要求主动扫描。但裁决警告称，强制扫描将破坏所有用户端到端加密，存在隐私泄露、未经授权图像泄露（如“The Fappening”事件）及政府过度干预的风险。法官承认儿童是当前法律框架下的“附带损害”。该案预计将上诉至第九巡回法院。
