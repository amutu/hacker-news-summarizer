## 1. 人体肠道中发现能增强肌肉力量的细菌

**原文标题**: Bacteria found in the human intestine capable of improving muscle strength

**原文链接**: [https://www.ugr.es/en/about/news/bacteria-found-human-intestine-capable-improving-muscle-strength](https://www.ugr.es/en/about/news/bacteria-found-human-intestine-capable-improving-muscle-strength)

格拉纳达大学、阿尔梅里亚大学和莱顿大学的研究人员发现，一种名为*Roseburia inulinivorans*的特定肠道细菌与肌肉力量和体能的改善有关。这一发现支持了“肠-肌肉轴”的概念。

研究分析了年轻人和老年人的粪便样本，发现这种细菌含量越高，握力和心肺能力越强。值得注意的是，携带该细菌的老年人握力比未携带者强29%。

在小鼠实验中，补充人类来源的*Roseburia*菌株使握力提升约30%，肌肉纤维增大，并引起肌肉组织有益的代谢变化。

关键发现是，这种细菌在老年人体内含量较低，表明其减少可能与年龄相关的肌肉流失有关。这为开发益生菌补充剂以帮助维持衰老过程中的肌肉力量提供了可能。

研究人员指出研究存在局限性，包括该细菌未在小鼠体内永久定植，且确切的因果机制需要进一步的长期研究。该成果发表于《肠道》期刊。

**主要特性：**
*   **大幅减少令牌：** 平均减少 65% 的输出令牌（范围从 22% 到 87%），从而实现更快的响应和更低的 API 成本。

*   **易于使用：** 安装后，可通过简单的命令触发，如 `/caveman` 或“像原始人一样说话”。
*   **有研究支持：** 引用了一项研究，表明强制简洁实际上可以提高模型在某些任务上的准确性。

**工作原理：** 该插件剥离冗长的措辞，但不会改变 AI 的底层推理逻辑。结果是直接、可操作的建议，没有多余的对话修饰。

**优势：** 用户在处理代码调试、架构解释和 PR 审查等技术查询时，可以获得更快、更便宜且通常更易读的响应，同时还带有一丝幽默感。

## 8. 一个（夜间版）Rust中的尾调用解释器

**原文标题**: A tail-call interpreter in (nightly) Rust

**原文链接**: [https://www.mattkeeter.com/blog/2026-04-05-tailcall/](https://www.mattkeeter.com/blog/2026-04-05-tailcall/)

本文详细介绍了作者在Rust中为Uxn CPU实现高性能尾调用解释器的过程，该实现利用Rust nightly版本的新`become`关键字实现了真正的尾调用优化。

该解释器通过采用"线程化代码"技术改进了先前较简单的Rust虚拟机——每条指令的实现都以直接跳转到下一条指令函数作为结束。这种方式使CPU状态保持在寄存器中，并使分支预测更高效，类似于手写汇编的实现。

核心创新在于使用`become`指令引导Rust编译器用下一个函数的栈帧替换当前栈帧，从而避免栈溢出并实现预期的跳转行为。通过复杂的宏为每个指令的包装函数生成模板代码。

性能表现存在差异：在ARM64架构（Apple M1）上，尾调用解释器性能优于作者手写的汇编代码；但在x86-64架构上，虽然超越了原始虚拟机，仍落后于汇编版本。作者分析生成的x86代码发现，相较于精心编写的汇编代码，编译器生成的代码优化不足（例如存在不必要的寄存器交换和溢出操作），这解释了性能差距。

该项目证明，通过尾调用技术，安全Rust能在虚拟机模拟中实现接近汇编的性能，但不同架构的编译器代码生成差异会影响最终结果。

**原文链接**: [https://github.com/salmanmohammadi/nanocode/discussions/1](https://github.com/salmanmohammadi/nanocode/discussions/1)

**Nanocode 文章摘要**

Nanocode 是一个开源库，用于在 TPU 上使用 JAX 从头开始训练类似 Claude Code 的编码助手。该项目借鉴了 nanochat 的训练方法，并实现了简化版的 Constitutional AI，以根据定义的 "SOUL" 文档来对齐模型，塑造其友好、随意且具代理性的人格。

关键步骤包括：
1.  **预训练与分词：** 模型在 FineWeb-Edu 和 The Stack-V2 代码数据（1:5 比例）的混合上进行预训练，优化了代码效率的分词，但牺牲了部分通用文本性能。
2.  **代理接口：** 模型设计为通过结构化的工具调用系统进行交互。它使用特殊令牌来模板化四个核心工具：`Read`、`Edit`、`Grep` 和 `Bash`，从而实现文件操作和命令执行。
