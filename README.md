## 1. 43万年前保存完好的木制工具是迄今发现的最古老文物。

**原文标题**: 430k-year-old well-preserved wooden tools are the oldest ever found

**原文链接**: [https://www.nytimes.com/2026/01/26/science/archaeology-neanderthals-tools.html](https://www.nytimes.com/2026/01/26/science/archaeology-neanderthals-tools.html)

**《发现43万年前保存完好的木制工具，刷新最古老记录》摘要**

考古学家在西班牙北部一处遗址发现了已知最古老的木制工具，其年代可追溯至约43万年前。这一发现极大地刷新了保存下来的木制技术的记录，此类文物极少能从如此久远的年代留存至今。

这些文物包括一根尖头挖掘棒和一件可能被磨钝的工具，发现于尼安德特人遗址拉奎瓦德洛斯托雷洪内斯一处浸水的泥炭层中。其异常完好的保存状态归功于泥炭沼泽独特、缺氧的环境条件。

分析表明，这些工具由黄杨木（一种致密的硬木）制成，并显示出复杂工艺的清晰证据。木材经过精心塑形，使用了诸如用石器刮削和雕刻等技术，随后可能还经过火烤硬化以制成耐用的尖端。

这一发现具有重要意义，原因如下：
*   它证明早期人类（可能是前尼安德特人或早期尼安德特人）已熟练掌握加工木材的技能——而木材这种易腐材料在此时期的考古记录中基本缺失。
*   这表明他们的技术能力和认知水平比仅从石器工具中先前所理解的更为先进。
*   尖头木棒的设计暗示其用于挖掘，可能是挖掘块茎、昆虫，甚至用于防御，这为了解他们的日常生存活动提供了直接线索。

这一发现对传统的"石器时代"标签提出了挑战，强调那很可能也是一个"木器时代"，有机材料在早期人类技术中扮演了核心角色，只是大多已随时间腐朽。

## 2. 棱镜

**原文标题**: Prism

**原文链接**: [https://openai.com/index/introducing-prism](https://openai.com/index/introducing-prism)

**《Prism简介》摘要**

OpenAI推出了Prism，这是一种评估AI模型安全性的新方法，特别关注其可能被滥用于制造生物威胁的潜在风险。该举措是OpenAI"准备框架"的一部分。

Prism解决的核心问题是，难以评估强大的AI模型能在多大程度上降低非专业人士获取危险生物材料相关信息和操作步骤的门槛。传统的人工专家评估方式缓慢、昂贵且难以规模化。

Prism的工作原理是使用一组AI"评判"模型，模拟恶意行为者利用AI助手实施生物威胁时可能采取的步骤。它测试模型在五个关键领域提供准确、可操作信息的能力：获取病原体、合成DNA、培养制剂、扩大生产和武器化/投放。AI评判员根据模型回答的正确性和实用性进行评分。

初步测试的关键发现是，当前最强大的模型（如GPT-4）与现有互联网资源相比，仅能带来**"轻微的提升"**。它们并未显著增加超越当前有决心的个人在网上所能找到信息之外的风险。然而，能够自动且大规模地进行此类评估本身是一项重大进步。

最终，Prism并非作为最终解决方案提出，而是作为一个可扩展的早期预警系统。它旨在帮助研究人员持续监控模型能力，识别模型进化过程中可能出现的风险激增，并在部署新系统之前为安全决策提供信息。相关结果和方法论已与美国政府及其他AI实验室共享。

## 6. OpenSSL：CMS AuthEnvelopedData解析中的栈缓冲区溢出

**原文标题**: OpenSSL: Stack buffer overflow in CMS AuthEnvelopedData parsing

**原文链接**: [https://openssl-library.org/news/vulnerabilities/#CVE-2025-15467](https://openssl-library.org/news/vulnerabilities/#CVE-2025-15467)

本文详细介绍了2026年1月27日披露的OpenSSL多个版本中发现的安全漏洞。这些问题的严重程度从中等到低不等，其中一项被评为高危。

最关键的漏洞（CVE-2025-15467，高危）是在使用AES-GCM等AEAD密码解析CMS AuthEnvelopedData消息时出现的栈缓冲区溢出。攻击者可通过发送包含超大初始化向量（IV）的特制消息触发此漏洞，可能在身份验证前导致崩溃或远程代码执行。

一项中等严重性问题（CVE-2025-11187）涉及PKCS#12文件中PBMAC1参数验证不当，可能在MAC验证期间导致栈缓冲区溢出或空指针解引用，引发拒绝服务或可能的代码执行。

同时披露的多个低危漏洞包括：
*   QUIC连接中`SSL_CIPHER_find()`函数的空指针解引用
*   `openssl dgst`命令行工具对大输入数据的静默截断
*   TLS 1.3证书压缩中的过量内存分配
*   `BIO_f_linebuffer`过滤器中的堆越界写入
*   使用底层OCB API调用时未验证的尾部字节
*   `PKCS12_get_friendlyname()`函数中的越界写入

受影响的OpenSSL版本包括3.6、3.5、3.4、3.3、3.0、1.1.1和1.0.2，每个版本均有对应的修复版本。受影响版本中的FIPS模块不受影响，因为问题存在于FIPS边界之外的代码中。建议用户升级至修复版本。
