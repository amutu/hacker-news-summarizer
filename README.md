作者在研究冷门的1994年世嘉Genesis游戏《梦幻之星复刻版》——即原版Master System游戏的再发行版本时，偶然看到一篇来自陌生网站的文章。该文章错误地声称这款游戏拥有升级后的图像、天气效果和昼夜循环系统。作者意识到这很可能是大语言模型典型的“幻觉”现象，并通过查询ChatGPT证实了这一猜测——后者同样提供了错误信息。

## 8. 阿姆斯特丹编译器套件

**原文标题**: Amsterdam Compiler Kit

**原文链接**: [https://github.com/davidgiven/ack](https://github.com/davidgiven/ack)

阿姆斯特丹编译器套件（ACK）是一个完整的多语言编译器工具链，最初由阿姆斯特丹自由大学开发。它包含针对ANSI C、Pascal、Modula-2和Basic的前端编译器，以及代码生成器和支持库，能够为广泛的历史和现代平台生成可执行文件，例如CP/M、MS-DOS、Linux（支持x86、68k、MIPS和PowerPC架构）以及PDP-11 V7 Unix等经典系统。

安装需要标准开发工具（ANSI C编译器、flex、yacc、GNU make、支持posix的Lua、Python 3.4+），并通过Makefile进行管理。主命令`ack`会根据文件扩展名自动选择语言，并支持使用`-m`标志进行跨平台编译。

尽管ACK已相当成熟，但其库支持有限（大致符合ANSI C标准），使用自有的目标文件格式，且发行版中的某些遗留组件可能无法编译。建议用户加入项目邮件列表以获取支持并报告问题。该软件采用类BSD许可证发布。

**原文标题**: A header-only C vector database library

**原文链接**: [https://github.com/abdimoallim/vdb](https://github.com/abdimoallim/vdb)

vdb是一个轻量级、仅头文件的C语言库，用于构建和查询向量数据库。它存储高维向量嵌入，并支持使用余弦距离、欧氏距离或点积距离度量进行最近邻搜索。

核心特性包括：采用无依赖的单文件设计（`vdb.h`），支持通过编译时标志启用可选的多线程功能，以及能够将数据库保存到磁盘或从磁盘加载。API提供了创建数据库、添加/删除带可选ID和元数据的向量，以及执行k近邻搜索的函数。

该库支持自定义内存分配器，并在启用多线程时使用读写锁，以实现并发搜索的同时使添加/删除操作互斥。它采用自定义二进制文件格式实现数据持久化，但元数据不会被保存。项目基于Apache 2.0许可证发布。

## 10. 代数效应的趣味之旅——从玩具示例到Hardcaml仿真

**原文标题**: Fun with Algebraic Effects – From Toy Examples to Hardcaml Simulations

**原文链接**: [https://blog.janestreet.com/fun-with-algebraic-effects-hardcaml/](https://blog.janestreet.com/fun-with-algebraic-effects-hardcaml/)

本文探讨了在OCaml 5中从使用单子（monads）转向代数效应（algebraic effects）的过程，特别是在Jane Street的Hardcaml仿真库中的应用。文章指出单子的两个主要缺点：它们通过需要普遍使用的语法糖和特殊库函数使代码可读性变得复杂，并且阻碍了未装箱类型（unboxed types）和局部模式（local mode）等高级OCaml特性的使用。

代数效应提供了一种更优雅的替代方案，允许计算暂停并将控制权交给处理器（handler），然后处理器可以恢复执行。这消除了对单子包装器的需求，从而产生了更简洁、更直接的代码。文章通过一个基于效应的基本算术运算计算示例，并与单子方法进行了对比。

核心应用在于Hardcaml的FPGA设计仿真中，其中效应管理数字电路与测试平台线程之间的同步。这展示了代数效应在并发原始目的之外的多样性，突显了其以单子无法实现的方式简化复杂、有状态交互的能力。
