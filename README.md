**MAI-Code-1-Flash 摘要**

微软推出了 **MAI-Code-1-Flash**，这是一种针对速度、效率和实际开发者工作流优化的新编程模型。该模型正逐步向 VS Code 中的 GitHub Copilot 个人用户推出。

该模型由微软端到端构建，使用清洁的授权数据，并专门针对 GitHub Copilot 生产环境进行训练。由于专注于真实环境而非仅仅基准测试，它能在智能化任务中更好地与周边开发者工具交互。

主要特性包括 **自适应思考**，对简单请求保持简洁响应，同时为复杂问题分配更多推理资源。这使得困难任务最多可减少 **60% 的令牌**，从而降低延迟和成本。

在与 Claude Haiku 4.5 的基准测试对比中，MAI-Code-1-Flash 在所有核心编码评估中均胜出，包括在 **SWE-Bench Pro 上领先 16 个百分点**（51.2% 对 35.2%）。它在指令遵循、数学、科学以及一项定制对抗性推理测试（准确率 85.8%）中也表现出色。

微软将该模型定位为“为开发者而非基准测试而生”，旨在以更好的令牌价值提供高质量的编程辅助。

**原文链接**: [https://microsoft.ai/news/introducing-mai-thinking-1/](https://microsoft.ai/news/introducing-mai-thinking-1/)

**MAI-Thinking-1摘要**

微软人工智能推出了**MAI-Thinking-1**，这是一个拥有350亿活跃参数（总计1万亿）的稀疏混合专家推理模型。该模型虽规模中等，但在软件工程领域（SWE-Bench Pro上媲美Claude Opus 4.6）和高等数学领域（AIME 2025达97.0%，AIME 2026达94.5%）均具有与领先模型竞争的实力。在盲评中，用户对其偏好度超过Claude Sonnet 4.6。

其核心差异在于采用**“爬山机”**理念：能力从零开始习得（无第三方蒸馏），训练使用经过商业许可的清洁数据（无AI生成的预训练数据），且从加速器到强化学习框架的整个技术栈均为自主研发。这确保了模型的可操控性、质量以及端到端优化。

该模型专为企业打造：支持25.6万token上下文、函数调用、多层指令及聊天补全API，并通过Microsoft Foundry提供安全保障。安全性被整合到与能力训练相同的奖励系统中，在拒绝回答与有用性之间取得平衡。

MAI-Thinking-1现已在Microsoft Foundry上提供私有预览，公开预览即将推出。它代表着迈向**人本超级智能**的一步，旨在服务人类而非取代人类。

## 7. 开放维修数据标准 – 开放维修联盟

**原文标题**: Open Repair Data Standard – Open Repair Alliance

**原文链接**: [https://openrepair.org/open-data/open-standard/](https://openrepair.org/open-data/open-standard/)

开放维修数据标准（ORDS）是一个用于收集和共享小型电器与电子设备维修数据的共享框架。其目标是通过整合不同社区维修团体的开放数据，识别全球、国家和区域的维修趋势，例如常见故障与修复方法。

该标准持续演进，最新版本0.3于2021年12月发布。主要改进包括新增维修状态、产品类别、合作伙伴产品类别、维修障碍和国家等字段，同时因数据质量问题移除了型号字段。标准收集三大类数据：**产品相关**（类别、品牌、生产年份）、**维修相关**（问题、维修状态、维修障碍）以及**活动相关**（ID、团体标识符、活动日期）。数据以开放知识共享许可协议进行共享。

开放维修联盟将成员团体的数据整合为ORDS格式，每半年在其网站上发布合并数据集。未来将聚焦于故障类型分类，并探索使用UNU-keys进行产品分类。

## 9. 即将入侵你网页浏览器的广告联盟

**原文标题**: The advertising cartel coming to your web browser

**原文链接**: [https://blog.zgp.org/the-advertising-cartel-coming-to-your-web-browser/](https://blog.zgp.org/the-advertising-cartel-coming-to-your-web-browser/)

**摘要：**

文章警告称，由Meta、谷歌、苹果和Mozilla联合开发的名为“归因等级1”的新型内置浏览器广告测量系统。该系统声称能在防止跨站识别的同时衡量广告效果（展示次数与转化次数），从而增强隐私保护。但作者认为，这是一个绕开用户同意和隐私法规的“卡特尔”，构建了双重标准体系：大科技公司的追踪默认开启，而竞争对手却必须遵守严格规则。

**文章指出了两大危害：**

1.  **高估大科技公司广告效果：** 该系统将销售功劳归因于搜索、社交和应用商店广告（漏斗底层），即使需求由其他渠道创造。这导致更多广告收入流向大科技公司，使合法出版商陷入困境，并损害地方新闻等民主机构。
2.  **激励风险更高的追踪行为：** 通过模糊广告与购买之间的关联，该系统使得利用机器学习清洗来自侵扰性行为（如智能电视麦克风）的数据变得更加容易，同时也让区分有效广告与“最后一次点击”的运气变得更加困难。

作者认为，隐私是一个集体性问题——无法通过个人选择退出解决——并且该体系会被用于游说，以抵制监管。作者呼吁叫停这一提案，或者至少确保用户能够通过浏览器扩展程序（如广告拦截器）管理归因追踪。文章最后指出，这并非不可避免，并将此状况与过去的行业合谋相提并论，敦促监管机构予以关注。
