**原文标题**: Open Chaos: A self-evolving open-source project

**原文链接**: [https://www.openchaos.dev/](https://www.openchaos.dev/)

**概述：**

Open Chaos 是一个实验性的、自治的开源项目，它没有中央维护者。其核心原则是：任何用户都可以提交拉取请求（PR），社区通过 openchaos.dev 上公开的、有时限的投票系统共同决定其命运。

项目的状态处于持续变化中，随着每一次合并而演进。网站界面会实时显示下一次计划合并事件的倒计时（“d --h --m --s”），突显了该流程自动化、限时的特性。它列出了“开放的 PR”并邀请访客“投票决定合并”，强调直接的民主参与。

其关键创新在于，用透明、算法化的治理模式取代了传统的维护者权威。所有决策——从错误修复到功能添加，甚至是恶意代码——都在预定的合并窗口期内由多数投票决定。这创造了一个鲜活的系统，其代码库完全由贡献者集体且常常不可预测的意愿所塑造，将混沌作为演进的核心机制。

## 4. ASCII驱动开发

**原文标题**: ASCII-Driven Development

**原文链接**: [https://medium.com/@calufa/ascii-driven-development-850f66661351](https://medium.com/@calufa/ascii-driven-development-850f66661351)

**“ASCII驱动开发”概要**

本文介绍了“ASCII驱动开发”（ADD）这一软件开发方法论，其核心理念是在编写代码之前，优先创建清晰的、基于文本的图表和文档。它被定位为一种务实的替代方案，以取代UML或复杂图表工具等更正式的方法论。

ADD的核心原则是使用简单的ASCII艺术——用诸如`-`、`|`、`+`和`/`等字符创建——直接在代码注释或文档文件中可视化地规划系统组件、数据流和架构。作者认为这种方法具有以下关键优势：

*   **简单与快速：** 除文本编辑器外无需任何特殊工具，减少了阻力，便于快速迭代想法。
*   **清晰与聚焦：** ASCII艺术的限制迫使开发者将复杂系统提炼出其核心部分，从而提升概念理解。
*   **持久性与可移植性：** ASCII图表是纯文本，使其易于版本控制、普遍可读，且不受专有工具过时的影响。它们与所描述的代码共存。

文章将ADD定位为并非取代所有文档，而是一种用于初始设计和沟通阶段的轻量级实用技术。其目标是确保开发者和团队对系统共享一个连贯的心智模型，从而从一开始就写出更清晰、更有意图的代码。它强调在实施之前通过视觉方式思考结构，以降低后期进行代价高昂的重构风险。

核心问题有两方面：首先，从技术层面控制人工智能的内在价值体系十分困难，这可能导致令人意外且担忧的行为；其次，社会对于人工智能应具备何种“正确”价值观尚未达成共识，使得使其与人类偏好保持一致本身就充满挑战。

结论是，若缺乏可靠且可预测的伦理框架，将难以信任人工智能承担日益重要的自主决策，这在该技术发展过程中构成了重大风险。

## 7. Bichon：一款轻量级、高性能的 Rust 电子邮件归档工具，配备 Web 界面。

**原文标题**: Bichon: A lightweight, high-performance Rust email archiver with WebUI

**原文链接**: [https://github.com/rustmailer/bichon](https://github.com/rustmailer/bichon)

Bichon是一款基于Rust开发的轻量级高性能邮件归档工具。与标准邮件客户端不同，它被设计为独立的服务器应用程序，专门用于从IMAP账户归档、索引和搜索历史邮件，不具备发送功能。

核心功能包括多账户同步、全邮件全文检索、支持IMAP与OAuth2认证，以及内置Web界面。它采用Tantivy引擎实现统一存储与检索，通过透明压缩和去重技术确保运行效率。

可通过Docker或预编译二进制文件快速部署。自0.2.0版本起，系统强制启用访问令牌认证机制，内置管理员用户并支持基于角色的访问控制。同时提供命令行工具支持导入现有邮件归档（EML/MBOX格式）。

Bichon采用开源协议（AGPLv3），以本地数据自主管理为核心原则，致力于为邮件归档与管理提供专业高效的解决方案。

## 8. 老鼠被拍到捕食飞行蝙蝠

**原文标题**: Rats caught on camera hunting flying bats

**原文链接**: [https://scienceclock.com/rats-caught-on-camera-hunting-flying-bats-for-the-first-time/](https://scienceclock.com/rats-caught-on-camera-hunting-flying-bats-for-the-first-time/)

德国北部的研究人员在石灰岩洞穴中记录到褐家鼠在半空中主动捕食蝙蝠，这是首次有记录的啮齿动物拦截飞行哺乳动物的证据。研究团队在塞格贝格卡尔克山洞穴使用红外和热成像摄像机，观察到老鼠从岩架上跃起捕捉蝙蝠，确认了13次捕杀行为，并发现了52具蝙蝠尸体的藏匿点。

这一行为展现了褐家鼠出人意料的敏捷性和行为可塑性，它们通常是杂食性食腐动物。老鼠采用了两种捕食策略：在洞穴入口处进行空中拦截，以及在地面攻击爬行的蝙蝠。它们很可能依靠胡须和听觉线索在近乎黑暗的环境中狩猎。

研究警告称，这种捕食行为构成了新的保护威胁，因为一小群老鼠可能在一个季节内消灭当地约7%的冬眠蝙蝠种群。欧洲蝙蝠已面临栖息地丧失和疾病的威胁，入侵性褐家鼠进入主要冬眠地点可能产生重大的生态影响。研究人员建议采取预防措施，在大型蝙蝠栖息地附近控制老鼠的存在。

## 10. Httpz – 适用于OxCaml的零分配HTTP/1.1解析器

**原文标题**: Httpz – Zero-Allocation HTTP/1.1 Parser for OxCaml

**原文链接**: [https://github.com/avsm/httpz](https://github.com/avsm/httpz)

**Httpz** 是专为 OxCaml 语言设计的高性能 HTTP/1.1 解析器与序列化器，旨在实现零堆内存分配。它利用 OxCaml 的无装箱记录和局部列表，将所有解析结果（如请求和头部）保持在栈上分配，从而大幅降低内存开销。

核心特性包括：基于大数组缓冲区的直接 I/O、完整的 HTTP/1.1 支持（方法、头部、分块编码），以及内置的异步静态文件服务器。其架构采用基于跨度的解析方式，通过偏移量和长度引用字符串，避免字符串复制，并复用预分配的 32KB 读取缓冲区。

性能基准测试显示，相较于基于 Eio 的解析器（httpe），Httpz 实现了显著提升：解析速度提高 2–3 倍，内存分配减少高达 829 倍（按字计数）。Httpz 仅需 209 纳秒即可解析最小请求，堆分配仅占用 3 个字，吞吐量达到每秒 1460 万请求。

该库包含一个可用于生产环境的静态文件服务器，支持 MIME 类型检测和目录保护等功能。通过 OxCaml 编译器安装，内置对比性基准测试工具。项目基于 ISC 许可证发布。
