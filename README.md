## 3. 回归地道设计

**原文标题**: Bring Back Idiomatic Design

**原文链接**: [https://essays.johnloeber.com/p/4-bring-back-idiomatic-design](https://essays.johnloeber.com/p/4-bring-back-idiomatic-design)

本文认为，现代网络软件已失去了桌面软件时代（如Windows 95-7）那种一致、直观的设计习惯。作者将“习惯性设计”定义为用户能瞬间理解的标准模式——例如“保持登录”的复选框。

过去，操作系统强制推行同质化界面，拥有统一的菜单、键盘快捷键和清晰的标签（文字优先于图标）。这使得软件易于学习和使用。然而，如今的网络应用却呈现出异质化界面。即使是Figma和Linear等优秀应用，也采用独特的设计、图标和快捷键，迫使用户不断重新学习交互方式。

作者将这种衰退归因于两个主要因素：1）向移动端过渡的笨拙过程，稀释了以桌面为中心的模式；2）前端开发（React、WebAssembly）的快速演进，工程师优先考虑新功能而非打磨标准化交互。

尽管如此，苹果和Substack等平台的成功表明，强大而具有明确主张的设计系统能创造连贯的“开箱即用”体验，这具有重要价值。文章最后为开发者提供了实用建议：遵循HTML/CSS和浏览器惯例，优先考虑清晰性而非美观性，使用文字而非模糊的图标，共同迈向更一致、更用户友好的界面未来。

## 10. 懒惰之危，失不再来

**原文标题**: The Peril of Laziness Lost

**原文链接**: [https://bcantrill.dtrace.org/2026/04/12/the-peril-of-laziness-lost/](https://bcantrill.dtrace.org/2026/04/12/the-peril-of-laziness-lost/)

本文认为，程序员“懒惰”的美德——即通过创建高效抽象以节省未来精力的驱动力——正受到现代开发趋势的侵蚀，尤其是大型语言模型的滥用。

文章首先赞扬了拉里·沃尔提出的“有益的懒惰”概念：通过艰苦的智力劳动构建简洁而强大的系统，以避免重复性工作。这种做法将使所有未来的开发者受益。然而，作者指出，这种理念正被一种虚假的勤奋文化（“奋斗崇拜”）所取代，而大型语言模型进一步加剧了这一趋势。

核心问题在于，大型语言模型天生缺乏人类所面临的时间有限性约束。它们生成代码时毫不考虑未来的复杂性，导致系统臃肿低效——文中幽默地引用了一个案例：某项目吹嘘其数万行代码均由大型语言模型生成，恰恰印证了这一问题。

作者总结道，尽管大型语言模型是强大的工具，但必须由人类主导的“有益的懒惰”来引导。它们应当用于解决技术债务、促进严谨性，并始终服务于长期目标：创建更简洁、更可组合的系统。时间这一根本的人类约束，才是最终催生优秀抽象的核心动力，而这是大型语言模型无法自行复制的纪律。
