## 1. 那么，你遇到了年龄限制。现在该怎么办？

**原文标题**: So, you've hit an age gate. what now?

**原文链接**: [https://www.eff.org/deeplinks/2026/01/so-youve-hit-age-gate-what-now](https://www.eff.org/deeplinks/2026/01/so-youve-hit-age-gate-what-now)

这篇来自电子前沿基金会（EFF）的文章是为遇到网络年龄验证要求的用户提供的指南。文章指出，尽管EFF反对此类强制要求，但它们已然生效，迫使用户在获取服务和保护隐私之间做出选择。

核心建议是尽可能提交最少的个人数据。指南评估了常见的验证方法：
*   **面部年龄估算：** 对边缘群体往往不准确。部分服务（如Yoti）会将您的照片上传至其服务器，而其他服务（如Private ID）则在您的设备上处理。
*   **证件上传（身份证）：** 需要同时证明年龄和身份，一旦泄露（如Discord曾发生的情况）会形成高风险的数据痕迹。
*   **信用卡或邮箱验证：** 即时风险低于身份证验证，但仍会破坏匿名性，并可能将您的活动关联到不同数据库。

文章针对主要平台提供了具体分析：
*   **Meta（Facebook、Instagram）：** 使用Yoti进行面部扫描（上传照片）或身份证验证。Meta声称会在30天后删除身份证信息。
*   **Google/YouTube：** 提供多种选项，优先选择侵入性较低的方法，如使用Private ID在设备上进行面部扫描、信用卡验证或通过第三方数据经纪商进行邮箱验证。
*   **TikTok：** 使用Incode进行身份证验证，该服务默认无限期保留图像，尽管TikTok声称会启动删除程序。

指南始终敦促用户提出关键问题：收集了哪些数据、谁有权访问、数据保留多久以及存在何种审计机制，并强调没有哪种方法对隐私和言论自由是完全安全的。

## 2. 为什么有些衣服洗后会缩水以及如何恢复原状

**原文标题**: Why some clothes shrink in the wash and how to unshrink them

**原文链接**: [https://www.swinburne.edu.au/news/2025/08/why-some-clothes-shrink-in-the-wash-and-how-to-unshrink-them/](https://www.swinburne.edu.au/news/2025/08/why-some-clothes-shrink-in-the-wash-and-how-to-unshrink-them/)

本文解释了衣物在洗涤时缩水的原因，并提供了一种可能逆转缩水的方法。缩水现象源于纺织纤维的自然特性。

棉、麻等天然纤维具有“记忆性”，在洗涤过程中受热、潮湿和机械搅动的影响时，往往会恢复原本卷曲的状态。这会使被拉伸的纤维松弛，导致织物收缩。羊毛的缩水方式不同，它通过称为“毡化”的过程实现，其表面鳞片状纤维会相互缠结。

聚酯等合成纤维经过设计更具稳定性和结晶度，因此抗缩水性远优于天然纤维。

为减少缩水，文章建议使用冷水、轻柔模式和低转速脱水。

若衣物已缩水，可尝试补救方法：按每升温水配一汤匙护发素或婴儿洗发露的比例浸泡衣物。护发素能润滑纤维，使您在平铺晾干前轻柔地将织物拉伸回原形。此法虽未必能完全逆转严重缩水，但常可使衣物恢复穿着功能。

**《“山间莫奈”——滑雪地图艺术家詹姆斯·尼胡斯作品概述》**

詹姆斯·尼胡斯是一位著名艺术家，他手绘了超过200幅滑雪场雪道地图，从韦尔到惠斯勒，定义了群山的视觉形象。他的职业生涯始于1987年，当时他接替了该领域的先驱比尔·布朗。尼胡斯以其独特的“平面立体”风格而备受赞誉，这种风格巧妙地将鸟瞰视角与风景画的艺术感相结合，使雪道既精确又诱人。

他创作过程一丝不苟，包括研究航拍照片、实地考察滑雪场，先用铅笔绘制精细草图，再用水粉和丙烯颜料作画。一幅地图的完成可能需要长达三周时间。他的标志性技法包括用“树木线”界定雪道，以及通过细致的明暗处理来描绘坡道陡峭度和阳光照射。

尼胡斯的作品因其艺术性和功能性而备受珍视，既能唤起怀旧之情，又能帮助滑雪者规划路线并憧憬下一次滑行。尽管数字地图日益兴起，他手绘的美学风格依然备受推崇。2021年出版的书籍《地图背后的人》以及一部纪录片巩固了他的传奇地位。现已正式退休的尼胡斯，已将技艺传授给他指导过的一小群艺术家，确保其独特而备受喜爱的风格，在未来岁月里继续为滑雪场指南增添光彩。

最初尝试使用Zig因与Swift异步函数不兼容而失败。改用Swift后，作者遇到了多重障碍：为较新的macOS版本配置Swift包、在ArgumentParser框架中正确处理异步函数，以及摸索文档匮乏的`TranslationSession` API。

关键的突破来自观看相关的WWDC视频，其中揭示了需要使用`NaturalLanguage`框架自动检测输入语言，以及通过`LanguageAvailability` API检查翻译模型是否已安装。尽管系统的右键翻译功能正常，但命令行工具需要通过系统设置手动下载特定语言模型，这一过程独立且令人困惑。

在解决了语言检测、模型安装检查和正确的async/await模式等问题后，作者最终成功构建了一个可用的工具。这次经历凸显了在苹果框架的主要应用场景SwiftUI之外进行开发的挑战——对于底层编程使用，相关文档往往稀少且不够直观。

**核心概念：**
*   **HTML 即语法：** 程序以有效的 HTML 编写。例如，`<div>` 标签可能充当 `print` 命令。
*   **声明式逻辑：** 控制流（如循环和条件判断）和数据绑定通过特殊的 HTML 属性实现，避免了传统的命令式语句。
*   **自托管与执行：** HyTags 解释器本身用 HyTags 编写，展示了该语言的能力。代码直接由此解释器执行。

**展示示例：** 文章很可能包含一个示例，例如一个“Hello, World!”程序或一个简单的计数器，完全使用类似 HTML 的标签和属性实现，展示了熟悉的 Web 语法如何定义应用逻辑。

**总体目的：** HyTags 似乎是一个思想实验和概念验证，旨在挑战标记语言与编程语言之间的传统界限。它探索了使用普遍理解的 HTML 范式进行通用编程的潜力，主要是为了展示一种非常规的语言设计方法，而非用于实际生产。该项目旨在引发关于语法、语言范式以及 HTML 灵活性的讨论。

**原文链接**: [https://github.com/ReturnInfinity/BareMetal-Cloud](https://github.com/ReturnInfinity/BareMetal-Cloud)

本文介绍了**BareMetal-Cloud**，这是一个仅为10KB的极简内核，专为运行云应用程序设计。它是BareMetal exokernel的精简版本，仅包含必要的驱动程序，仅占用4MB内存，其余内存全部留给应用程序负载。
